  </section>1f:T41c,<p>Every interactive dashboard you’ve used is built on an event bus:</p>
<pre><code>click → mutate the crossfilter → emit &quot;change&quot; → listeners redraw
</code></pre>
<p>The filter is the source of truth; history, if it exists at all, is a bolted-on side-effect
watching the changes go by — and it can drift. The trail runs the same wire <strong>backwards</strong>:</p>
<pre><code>click → append to the log → notify &quot;the log moved&quot;
      → re-derive the filter from the fold — fresh, every time
</code></pre>
<p>In an event bus, the click updates the filter and history watches. In a trail, <strong>the click updates
history and the filter is watching.</strong> That one inversion buys three things for free: the record can
never desync from the dashboard, because the dashboard is <em>derived</em> from the record; replay and
time travel are just folding to an older step, a consequence rather than a feature; and there are
no feedback loops, because a re-render is a read — only a hand or an agent can write.</p>
